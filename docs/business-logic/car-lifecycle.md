## حالات السيارة (Car.status)

| الحالة | اللون | المعنى | الإجراءات |
|---|---|---|---|
| `DRAFT` | رمادي | مُدخلة غير معروضة | تعديل، نشر، حذف |
| `FOR_SALE` | أزرق | معروضة (ثابت أو سوم) | تعديل، إيقاف، تسجيل طلب |
| `AUCTION` | بنفسجي | معروضة كمزاد | قبول عرض، إيقاف |
| `RESERVED` | برتقالي | محجوزة لعميل | متابعة الصفقة |
| `SOLD` | أخضر | تمّ البيع | عرض فقط |

## مراحل طلب العميل (CarRequest.status)

```
PENDING → RESERVED → WAITING_PAYMENT → OWNERSHIP_TRANSFER → COMPLETED
                                                           ↘
                                              REJECTED / CANCELLED
```

| المرحلة | يعني | حالة السيارة |
|---|---|---|
| `PENDING` | طلب جديد لم يُرد عليه | FOR_SALE / AUCTION |
| `RESERVED` | قُبل الطلب، حُجز العميل | RESERVED |
| `WAITING_PAYMENT` | بانتظار دفع المبلغ | RESERVED |
| `OWNERSHIP_TRANSFER` | جاري نقل الملكية | RESERVED |
| `COMPLETED` | اكتمل البيع | SOLD |
| `REJECTED` | رُفض الطلب | FOR_SALE (يعود) |
| `CANCELLED` | ألغى العميل | FOR_SALE (يعود) |

**مهم:** عند تغيير حالة الطلب، تتغير حالة السيارة تلقائياً في نفس الـ transaction.  
عند RESERVED: يُنشأ تلقائياً سجل `Customer` إن لم يكن موجوداً.

## وضع العرض (Car.displayMode)

| الوضع | الوصف |
|---|---|
| `FIXED_PRICE` | سعر ثابت واضح |
| `SOUM` | سعر عرض + حد أدنى سري للقبول |
| `AUCTION` | مزاد بسعر افتتاح، ينتهي بوقت محدد |

---

## نوع المزاد (Car.auctionType)

| النوع | الوصف |
|---|---|
| `PUBLIC` | مزاد عام — يظهر في السوق |
| `PRIVATE` | مزاد خاص — رابط سري فقط (slug عشوائي) |
| `NONE` | ليس مزاداً |

## Timeline (Audit Log)

كل حدث يُسجَّل تلقائياً في `car_timeline` — لا تعديل ولا حذف أبداً.

| نوع الحدث | الوصف |
|---|---|
| `CAR_CREATED` | إدخال السيارة |
| `STATUS_CHANGED` | تغيير الحالة (from/to/displayMode/action) |
| `PRICE_CHANGED` | تغيير السعر |
| `FIELD_UPDATED` | تعديل أي حقل |
| `IMAGE_UPLOADED` | رفع صورة |
| `DOCUMENT_UPLOADED` | رفع مستند |
| `NOTE_ADDED` | إضافة ملاحظة |
| `SALE_REGISTERED` | تسجيل بيع |

## صفحة التفاصيل — التبويبات

1. **التفاصيل** — كل بيانات السيارة + مواصفات + بيانات المزاد/السوم
2. **المالية** — تكاليف الشراء، الإضافية، سعر البيع، الربح المتوقع
3. **المزايدات** — (يظهر عند AUCTION فقط) قائمة مرتبة + إحصائيات
4. **عروض السوم** — (يظهر عند SOUM فقط) تاريخ التفاوض مع إبراز المقبول
5. **السجل** — timeline كامل بتسميات بشرية
6. **المستندات** — كل الملفات المرفوعة
7. **الحوادث/موجز** — تقرير موجز إن وُجد

## نشر/إيقاف السيارة

**النشر:** `POST /api/v1/cars/[id]/publish` — يأخذ `mode` (FIXED_PRICE / SOUM / AUCTION)  
**الإيقاف:** `POST /api/v1/cars/[id]/unpublish` — يُعيد السيارة لـ DRAFT + يُسجّل في التايم لاين

## معرّف السيارة العام (carPublicId)

كل سيارة تحمل معرفاً عاماً فريداً على مستوى المنصة بالكامل:

- **الصيغة:** `CS` + آخر رقمين من السنة + 6 أرقام متسلسلة = **10 خانات**
- **مثال:** `CS26000001` (السيارة الأولى في 2026)
- **إعادة العداد:** كل سنة يبدأ العداد من 1 (`CS27000001` في 2027)
- **الفرق عن carRefNumber:** `carRefNumber` تسلسل داخل المعرض الواحد، أما `carPublicId` فهو تسلسل على مستوى المنصة بالكامل

### توليد الـID
في `repositories/car.repository.ts` → `generateCarPublicId()`:
1. يحدد السنة الحالية (`26` من 2026)
2. يجد أعلى تسلسل لهذه السنة عبر `findFirst` مرتباً تنازلياً
3. يضيف 1 ويولّد الـID الجديد
4. `UNIQUE` constraint في DB يضمن عدم التكرار
