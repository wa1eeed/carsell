## المرحلة ١ — Core MVP (عند الإطلاق)

### أنواع الحسابات والـ KYC

| نوع الحساب | المستندات المطلوبة | الميزات |
|---|---|---|
| وكالة / موزع | سجل تجاري + ترخيص وكالة | كل الميزات + لوحة الوكيل |
| معرض / صالة | سجل تجاري + ترخيص تجارة سيارات | كل الميزات |
| فرد — مواطن | هوية وطنية | بيع وشراء، بعض الميزات مقيّدة |
| فرد — مقيم | إقامة سارية | نفس المواطن بقيود إضافية |
| فرد — زائر | جواز سفر | بيع محدود، شراء فقط |
| شركة (تأمين/بنك) | سجل تجاري + ترخيص نشاط | لوحة مخصصة، بيع وشراء في السوق |

```prisma
enum AccountType {
  AGENCY         // وكالة / موزع
  SHOWROOM       // معرض / صالة
  INDIVIDUAL     // فرد
  COMPANY        // شركة (تأمين، بنك، إلخ)
}

enum IndividualType {
  CITIZEN        // مواطن
  RESIDENT       // مقيم
  VISITOR        // زائر
}

enum CompanyType {
  INSURANCE
  BANK
  FINANCING
  OTHER
}
```

### حقول السيارة الكاملة

**حقل نوع السيارة (موسّع):**
```
NEW           ← جديدة من الوكالة
USED          ← مستعملة عادية
USED_QUALIFIED← مستعملة مؤهلة (هامش ربح)
UNDER_CUSTOMS ← تحت الجمارك / بطاقة جمركية
```

**حقل نوع اللوحة (موسّع):**
```
PRIVATE       ← خاصة
TAXI          ← أجرة
TRANSPORT     ← نقل
DIPLOMAT      ← دبلوماسية
EXPORT        ← صادرة (لوحة تصدير)
CUSTOMS       ← حمركية
```

**حقول انتهاء الصلاحية (اختيارية — تُستبدل برباط يقين):**
```
registrationExpiry   ← انتهاء الاستمارة
inspectionExpiry     ← انتهاء الفحص
insuranceExpiry      ← انتهاء التأمين
insuranceProvider    ← شركة التأمين
insurancePolicyNo    ← رقم الوثيقة
```

**حقل وقت التواجد في المعرض:**
```
enteredShowroomAt    ← تاريخ الدخول للمخزون
→ عداد تلقائي يحسب عدد الأيام منذ الإدخال
```

### بيع بالتمويل

```typescript
interface FinancingDetails {
  financeProvider: string     // اسم شركة التمويل
  financeRate: number         // نسبة الرسوم %
  financeAmount: number       // المبلغ الممول
  financeFees: number         // رسوم التمويل (تُضاف للسعر)
  totalWithFees: number       // السعر الإجمالي للمشتري

}
```

آلية التحصيل: شركة التمويل تدفع للمنصة، المنصة تحوّل للبائع بعد خصم رسومها (إذا كانت صفقة عبر CarLink Market).

### المزاد والبيع المتعدد الأشكال

```
نوع العرض:
  FIXED_PRICE     ← سعر مباشر
  AUCTION_PUBLIC  ← مزاد عام (يراه كل المستخدمين)
  AUCTION_PRIVATE ← مزاد خاص (يراه عملاء البائع فقط)
  SOUM            ← على السوم (سعر قابل للتفاوض)

مسار السعر المباشر:
  عرض سعر → عربون حجز → مسار المعاينة → إتمام البيع

مسار المزاد:
  فتح المزاد → مدة زمنية → أعلى سعر → دفع عربون → إتمام
```

### نظام الأسعار (Pricing Engine)

كل عملية في المنصة لها سعر قابل للتخصيص:

```typescript
interface PricingRule {
  id: string
  key: string              // 'listing_fee' | 'sale_commission' | 'inspection_fee' | ...
  label: string            // النص المعروض للمستخدم
  amount: number
  type: 'fixed' | 'percentage'
  appliesTo: 'showroom' | 'individual' | 'provider' | 'all'
  isActive: boolean
  vatIncluded: boolean
}
```

الأدمن يتحكم في كل سعر من لوحته — تفعيل/تعطيل/تغيير النص.

## المرحلة ٢ — Growth (بعد الإطلاق)

### كوبونات الخصم
```prisma
model Coupon {
  id           String    @id @default(uuid())
  code         String    @unique
  type         String    // percentage | fixed
  value        Decimal   @db.Decimal(10, 2)
  appliesTo    String    // subscription | service | wallet_topup
  maxUses      Int?
  usedCount    Int       @default(0)
  expiresAt    DateTime?
  isActive     Boolean   @default(true)
  createdAt    DateTime  @default(now())
  @@map("coupons")
}
```

### المحفظة الإلكترونية
```prisma
model Wallet {
  id          String    @id @default(uuid())
  userId      String    @unique
  balance     Decimal   @default(0) @db.Decimal(12, 2)
  // الرصيد يُشحن عبر تابي / تمارا / مدى
  // يُستخدم لدفع اشتراكات، خدمات، عربون
  @@map("wallets")

model WalletTransaction {
  id          String    @id @default(uuid())
  walletId    String
  type        String    // topup | deduct | refund
  amount      Decimal   @db.Decimal(10, 2)
  reference   String?   // رقم العملية / سبب الخصم
  createdAt   DateTime  @default(now())
  @@map("wallet_transactions")
}
```

### لوحة الورش ومزودي الخدمة
```
مزود خدمة ثابت:   ورشة / مركز فحص / مستودع
مزود خدمة متنقل:  ميكانيكي متنقل / خدمة نقل سيارات

خصائص المزود:
- المدينة / المنطقة
- أنواع الخدمات
- التقييم
- التسعيرة
- فاتورة الخدمة (اسم الخدمة، الرسوم، المبلغ الإجمالي)

الإسناد:
- طلب عميل على سيارة → يُبثّ لمزودين في نفس المدينة
- المزود يقبل أو يرفض
```

### النشر التلقائي على السوشال ميديا
```
الأدمن يحدد:
  - قالب البوست (صورة + نص)
  - السيارات المؤهلة (مضافة حديثاً / بسعر معين / براند معين)
  - الجدولة (فوري / مجدول)
  - المنصات (تويتر، إنستغرام، سناب)

البائع يتحكم:
  - تلقائي أو يدوي
  - رابط تفاصيل السيارة مُرفق في البوست
```

### عداد الوقت وPipeline البيع
```
لكل سيارة:
  - عدد الأيام في المعرض (منذ enteredShowroomAt)
  - عدد الأيام منذ آخر تأجير (للسيارات المؤجّرة)
  - إشعار تلقائي عند تجاوز حد معين (مثلاً 30 يوم)

Pipeline البيع:
  مسودة → معروضة → مهتم → معاينة → عربون → مباعة
```

## المرحلة ٣ — Pro Features

### وكيل الذكاء الاصطناعي
```
حدود الاستخدام:
  - عدد المحادثات / الشهر (Basic: 50, Pro: 500, Enterprise: ∞)
  - رصيد قابل للشحن من المحفظة

ما يفعله الوكيل:
  - الإجابة على استفسارات المشترين
  - اقتراح سيارات بناءً على السلوك والبحث
  - تسعير ذكي عبر واتساب / popup / إيميل
  - تقارير أداء للبائع

قسم الوكيل:
  - إعداد شخصية الوكيل (اسم، نبرة)
  - قاعدة معرفة (FAQs + تفاصيل المخزون)
  - سجل المحادثات
```

### تصميم البوستات
```
أدوات التصميم:
  - قوالب إنستغرام / سناب / تيك توك
  - بيانات السيارة تُملأ تلقائياً
  - رابط صفحة السيارة مُرفق

التسعير:
  - رصيد يُخصم من المحفظة لكل بوست
  - صفحة قياس الاستخدام (عدد البوستات، الرصيد المستهلك)
```

### الطرح الاحترافي — Hardware Bundle
```
باقة Showroom Pro:
  سعر الباقة السنوية: X ريال
  + تابلت مناسب للمعرض: Y ريال (مخفّض)
  = المجموع: X+Y بخصم Z%

مثال:
  الباقة: 8,400 ريال/سنة
  + تابلت Samsung Tab A9: 800 ريال (بدلاً من 1,200)
  = 9,200 ريال (وفر 400)
```

### موديول التأجير
```
ربط مع: علم منتج (Alam Manal) — مزود خدمة تأجير
حقول إضافية للسيارة:
  - متاح للتأجير: نعم/لا
  - سعر اليوم / الأسبوع / الشهر
  - الحد الأدنى لأيام التأجير
  - آخر تأجير / عدد أيام الوقوف
```

## قرارات المنتج — Product Decisions

### نظام التأجير — مؤجّل بوعي

**القرار:** نظام التأجير يُبنى في المرحلة القادمة بعد استقرار نظام البيع.

**السبب:**
- نظام البيع هو الأساس — كل بنية البيانات والمدفوعات والتايم لاين مبنية حوله
- التأجير له منطق مختلف كلياً (عقود، مدد، صيانة دورية، تتبع مسافة)
- دمجهما مبكراً يزيد التعقيد دون قيمة في المرحلة الأولى

**ما يُحضَّر له الآن:**
- حقل `listedForRent: Boolean` على جدول `Car` — placeholder فقط، غير مفعّل
- لا UI، لا API، لا pricing لنظام التأجير حتى يستقر البيع

**عند البناء لاحقاً:**
- موديول مستقل `RentalModule` في route group منفصل `(rental)/`
- جداول جديدة: `RentalContract`, `RentalPeriod`, `RentalReturn`
- ربط مع علم منتج (Alam Manal) كمزود خارجي
